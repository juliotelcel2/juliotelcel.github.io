Creacion de un blog con github page

15 May 2016

Github pages es una plataforma de Sitios web donde se alojan los proyectos directamente desde el repositorio de GitHub ; Sólo hay que editar , subir y los cambios son en vivo .Ghost es una plataforma de publicación de código abierto que está muy bien diseñado , fácil de usar y libre para todo el mundo . pero hay que realizar una serie de pasos para poder ver la publicacion de un sitio.

https://upload.wikimedia.org/wikipedia/commons/thumb/f/fa/Ghost-Logo.svg/2000px-Ghost-Logo.svg.png

/home/julio/Imágenes/ghostimagen.png

Cuando entramos a github y creamos el repositorio, debe de ser con el nombre de nuestro usuario en github para no tener conflictos con la dirección web

Para utilizar git hub pages debemos tener instaladas 4 herramientas

python
node
pip(se debe actualizar a la version reciente)
buster

debo descargar ghots a mi escritorio.descomprimir el zip y crear la carpeta ghost.

para instalar en modo root el comando es sudo -i luego cd /y navegar al directorio donde tengo ghost,un vez este en la carpeta ghost por teminal le doy a npm install --produccion

luego se escribe el comando npm start para crear la ip, ella se escucha en el area local para entrar en la configuracion del blog entra a la direccion 127.0.0.1:2368/ghost

http://primbs.de/wp-content/uploads/2014/09/2014-09-01_10-35-06.png

En otra ventana del terminal navegamos hacia la carpeta ghost y luego escribimos el comando buster generate para que me genere los archivos stactic que se suben al github pages

luego hay que navergar a la carpeta static para subirla a github page para que genere los cambios...antes de subir debemos hacer git init,git add,git commit -m,luego debemos conectarnos por ssh.

para conectarme por ssh escribimos el comando git remote add origin "direccion_ssh"

y luego git push origin master....

Revisamos los cambios en github y nos metemos en edit para editar la direccion http://usuario.github.io y salvamos.

luego entramos con la direccion local 127.0.0.1:2368/ghost para el setup configuracion de la cuenta y el blog.
