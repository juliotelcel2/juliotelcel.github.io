Creación de un blog con Github Pages.

15 May 2016

Github pages es una plataforma de Sitios web donde se alojan los proyectos directamente desde el repositorio de GitHub; Sólo hay que editar, subir y los cambios son en vivo . Ghost es una herramienta de publicación de código abierto que está muy bien diseñado, fácil de usar y es libre para todo el mundo; pero hay que realizar una serie de pasos para poder realizar publicación de un sitio.

Cuando entramos a Github y creamos el repositorio, debe de ser con el nombre de nuestro usuario en Github para no tener conflictos con la dirección web. Para utilizar git hub pages debemos tener instaladas 4 herramientas:

python.
node.
pip(se debe actualizar a la version reciente).
buster.

Debemos descargar ghots al desktop de nuestro equipo, descomprimir el zip y crear la carpeta ghost.

Para instalar en modo root debemos teclear en la terminal el comando sudo -i luego cd / y navegar al directorio donde tengo ghost, una vez este en la carpeta ghost, en la terminal le doy a npm install --production.

Luego se escribe el comando npm start para crear la ip, ella se escucha en el area local para entrar en la configuracion del blog, entra a la direccion 127.0.0.1:2368/ghost.

En otra ventana del terminal, navegamos hacia la carpeta ghost y luego escribimos el comando buster generate , para que me genere los archivos stactic que se suben al github pages.

Luego hay que navegar a la carpeta static para subirla a github page para que genere los cambios...antes de subir debemos hacer la iteracion basica de git:
git init
git add -A
git commit -m"el cambio echo"

Despues hay que conectarnos por ssh ; para estos escribimos en la consola el comando git remote add origin "direccion_ssh" y luego git push origin master.

Revisamos los cambios en github y nos metemos en edit para editar la dirección usuario.github.io y salvamos.

Luego entramos con la dirección local 127.0.0.1:2368/ghost para el setup configuracion de la cuenta y el blog.
